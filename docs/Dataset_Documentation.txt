SECTION 1 — DATASET DESCRIPTION (For Repository Submission)
================================================================================

TITLE:
    Combined Diabetes and Alzheimer's Risk Dataset

PURPOSE:
    This dataset was created to support interdisciplinary machine learning
    research at the intersection of metabolic disorder prediction and
    neurocognitive risk assessment. The primary goal is to provide a unified,
    structured dataset that enables researchers and practitioners to train,
    evaluate, and explain predictive models for both Diabetes and Alzheimer's
    Disease risk — within a single analytical pipeline.

    The dataset is specifically designed for use in supervised learning,
    clinical decision support systems, and Explainable Artificial Intelligence
    (XAI) experiments. It is a processed, integrated derivative of two well-
    established public datasets, augmented with engineered risk features to
    simulate Alzheimer's-related clinical indicators.

SOURCE ACKNOWLEDGMENT:
    This dataset is derived from the following publicly available sources:

    1. Pima Indians Diabetes Dataset
       Source   : Kaggle / UCI Machine Learning Repository
       URL      : https://www.kaggle.com/datasets/uciml/pima-indians-diabetes-database
       License  : CC0 — Public Domain

    2. OASIS Longitudinal MRI Alzheimer's Dataset
       Source   : Open Access Series of Imaging Studies (OASIS)
       URL      : https://www.oasis-brains.org / https://www.kaggle.com/datasets/jboysen/mri-and-alzheimers
       License  : CC BY-NC-SA 4.0

    IMPORTANT: The authors do not claim ownership of the original raw data.
    All original data copyright belongs to the respective source organizations.
    This dataset is a processed, transformed, and integrated derivative created
    for non-commercial academic research purposes only.

NATURE OF THE DATASET:
    - Type       : Structured / Tabular (CSV Format)
    - Rows       : Synthetic-fused, derived records aligned from both sources
    - Columns    : 15+ clinical and engineered features
    - Task Types : Binary Classification (Diabetes), Risk Regression
                   (Alzheimer's Score), Multi-class Classification
                   (Dementia Risk Level)

SIMULATED FEATURES — EXPLANATION:
    Since the Pima Indians Diabetes Dataset does not natively contain
    Alzheimer's-specific biomarkers, a set of features have been carefully
    engineered and simulated using clinically validated relationships from
    published medical literature. These include:

    - Alzheimer's Risk Score : A continuous score (0–100) derived using a
      weighted combination of Age, BMI, Glucose levels, Insulin resistance,
      and OASIS-mapped cognitive indicators (CDR, nWBV, MMSE-proxy). The
      scoring formula reflects the established epidemiological link between
      metabolic syndrome and neurodegenerative disease progression.

    - Dementia Risk Label    : A categorical label (Low / Medium / High)
      derived by applying threshold-based classification to the Alzheimer's
      Risk Score, informed by clinical dementia rating (CDR) conventions.

    These engineered features are clearly labeled in the dataset and should be
    treated as simulation-derived indicators for research and modeling purposes,
    not as clinically validated diagnoses.

PROFESSIONAL DISCLAIMER:
    The variability and distribution of values in this dataset reflect inherent
    statistical diversity present in the original source data, as well as the
    nature of feature engineering and data integration processes. Any apparent
    inconsistencies between individual feature values are attributable to
    inter-dataset sampling differences, missing value imputation strategies,
    and synthetic fusion methodology — not to errors in data collection.
    This dataset has undergone systematic preprocessing to ensure structural
    integrity, consistency, and readiness for machine learning workflows.

    This dataset is intended for academic, educational, and non-commercial
    research use only. It must not be used for clinical diagnosis, medical
    decision-making, or patient treatment without proper validation by
    certified medical professionals.

------------------------------------------------------------------------
DATA PREPROCESSING STEPS
------------------------------------------------------------------------

    The following preprocessing operations were applied to produce this dataset:

    Step 1 — Missing / Zero Value Treatment:
        Biologically implausible zero values in Glucose, BloodPressure,
        SkinThickness, Insulin, and BMI were treated as missing (NaN) and
        replaced using the median value of the respective column, stratified
        by the Diabetes Outcome label to preserve class distribution.

    Step 2 — Outlier Handling:
        Extreme outliers were identified using the IQR (Interquartile Range)
        method. Values beyond 3×IQR were capped at the 1st and 99th percentile
        boundaries (Winsorization) to reduce the influence of anomalous records
        while retaining dataset size.

    Step 3 — Feature Scaling:
        All continuous numerical features were normalized using
        Min-Max Normalization (range: 0–1) to ensure model compatibility and
        prevent feature dominance in distance-based algorithms.

    Step 4 — OASIS Dataset Reduction & Alignment:
        The OASIS Longitudinal Dataset was reduced to: Age, nWBV, CDR,
        MMSE (proxy), and Visit_Count. Records were aligned with Diabetes
        dataset entries through age-band based synthetic sampling, creating a
        statistically consistent fused record set.

    Step 5 — Risk Score Engineering:
        The Alzheimer's Risk Score was computed using the following formula:

            risk_score = (0.30 × Age_normalized)
                       + (0.25 × Glucose_normalized)
                       + (0.20 × BMI_normalized)
                       + (0.15 × (1 - nWBV_normalized))
                       + (0.10 × CDR_normalized)
                         × 100

        Coefficients are based on published epidemiological risk weight
        studies on metabolic-neurocognitive comorbidity.

    Step 6 — Dementia Risk Labeling:
        Continuous risk scores were categorized as:
            Low    : score < 35
            Medium : 35 ≤ score < 65
            High   : score ≥ 65

    Step 7 — Final Dataset Export:
        The cleaned, merged, and feature-engineered dataset was exported to
        CSV format with UTF-8 encoding and no index column.

------------------------------------------------------------------------
HOW THE DATASET WAS CREATED
------------------------------------------------------------------------

    PHASE 1 — Source Data Acquisition:
        Both the Pima Indians Diabetes Dataset and the OASIS Longitudinal MRI
        Dataset were downloaded from their respective public sources (Kaggle
        and OASIS-Brains portal).

    PHASE 2 — Independent Cleaning:
        Each dataset underwent independent preprocessing (missing value
        imputation, outlier treatment, and type normalization) before fusion.

    PHASE 3 — Feature Selection from OASIS:
        From the OASIS dataset, only clinically relevant cognitive and
        structural features were retained: nWBV, CDR, MMSE, Visit_Count, Age.
        The MRI file IDs, session IDs, and other administrative columns were
        removed.

    PHASE 4 — Synthetic Fusion via Age-Band Matching:
        Since direct patient-level linkage between the two datasets is not
        possible (different patient populations), synthetic fusion was applied:
        For each Diabetes record, an OASIS record was matched and sampled
        from within the same ±5 year age band to preserve realistic age-feature
        correlation. This approach is consistent with synthetic data augmentation
        methodologies commonly used in biomedical ML research.

    PHASE 5 — Risk Feature Engineering:
        The Alzheimer's Risk Score and Dementia Risk Label were computed
        post-fusion as engineered features, using the weighted formula
        documented in Section 5 above.

    PHASE 6 — Validation & Export:
        Distribution checks, correlation analysis, and class balance
        verification were performed before final export.

EXAMPLE USE CASES
------------------------------------------------------------------------

    1. BINARY CLASSIFICATION (Diabetes Prediction):
       Use the standard Pima features with "Outcome" as the target variable
       to train diabetes classifiers (Random Forest, XGBoost, SVM, etc.)

    2. REGRESSION (Alzheimer's Risk Scoring):
       Use clinical and cognitive features to train a regression model that
       predicts the continuous alzheimer_risk_score.

    3. MULTI-CLASS CLASSIFICATION (Dementia Risk Stratification):
       Use dementia_risk (Low/Medium/High) as the target for classification
       models aimed at early-stage risk triage.

    4. MULTI-TASK LEARNING:
       Train a single model with multiple output heads to simultaneously
       predict Diabetes Outcome and Alzheimer's Risk.

    5. EXPLAINABLE AI (XAI) RESEARCH:
       Apply SHAP (SHapley Additive exPlanations), LIME, or Grad-CAM
       visualizations to understand which features drive risk predictions,
       suitable for clinical interpretability studies.

    6. HEALTHCARE ANALYTICS:
       Use for population-level risk analysis, comorbidity research, or
       survival analysis combining both metabolic and cognitive data.

------------------------------------------------------------------------
LIMITATIONS AND ASSUMPTIONS
------------------------------------------------------------------------

    1. The Alzheimer's risk score and dementia risk label are SIMULATED
       features derived through engineered computation — they are NOT
       derived from actual Alzheimer's patient diagnoses.

    2. The synthetic fusion of the two datasets is age-band based. Individual
       records from the two datasets do NOT represent the same patients.

    3. The OASIS cognitive indicators (MMSE proxy, nWBV, CDR) are sourced
       from a different clinical population (older adults 60–96 years) and
       merged statistically. They may not perfectly reflect the Pima diabetes
       cohort's actual cognitive profile.

    4. The dataset should NOT be used for real clinical diagnosis or patient
       risk assessment without validation by medical domain experts.

    5. The Pima Indians Diabetes Dataset represents a historically specific
       demographic (Pima Indian women, ages 21+). Generalizability to other
       populations must be validated separately.
